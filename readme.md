# Демонстрационный плагин

## Что это?

Это демонстрационный плагин, который показывает все возможности системы плагинов через удобный веб-интерфейс. Он помогает понять, что могут делать плагины и как они работают.

## Зачем он нужен?

Плагин создан для того, чтобы вы могли:

- Увидеть все возможности системы плагинов в действии
- Понять, как плагины взаимодействуют с системой
- Изучить примеры использования различных функций
- Протестировать работу плагинов в безопасной среде

## Установка

Перед использованием плагин нужно установить. Выполните команду:

```bash
tg pkg add https://github.com/seniorGolang/tgp-demo
```

После установки плагин будет доступен для использования.

## Как использовать?

### Запуск плагина

Просто выполните команду:

```bash
tg demo serve
```

Плагин запустит веб-сервер на порту 8080 (по умолчанию). Вы можете указать другой порт:

```bash
tg demo serve --addr :3000
```

### Открытие интерфейса

После запуска откройте в браузере:

```
http://localhost:8080
```

Вы увидите интерактивную страницу с различными разделами, каждый из которых демонстрирует определенную возможность плагинов.

## Что можно увидеть?

Веб-интерфейс содержит несколько разделов, каждый из которых показывает конкретную функцию:

### 1. Работа с данными

Показывает, как плагины получают и передают данные между собой.

### 2. Логирование

Демонстрирует различные уровни сообщений, которые плагины могут отправлять в консоль.

### 3. Работа с файлами

Показывает, как плагины могут создавать, читать и изменять файлы в разрешенных директориях.

### 4. HTTP запросы

Демонстрирует, как плагины могут отправлять запросы к внешним сервисам.

### 5. Веб-сервер

Показывает, что плагины могут запускать собственные веб-серверы и обрабатывать запросы.

### 6. Выполнение команд

Демонстрирует, как плагины могут запускать системные команды (например, `go version`, `uname`).

### 7. План выполнения

Визуализирует последовательность выполнения плагинов в виде графа.

### 8. Информация о системе

Показывает, как плагины могут получать информацию о системе, на которой они выполняются.

### 9. Переменные окружения

Демонстрирует доступ к переменным окружения системы.

### 10. Интерактивный выбор

Показывает, как плагины могут запрашивать выбор пользователя через консоль.

### 11. Хеширование файлов

Демонстрирует загрузку файлов и вычисление их хешей.

### 12. Фоновые задачи

Показывает, как плагины могут запускать долгосрочные задачи, которые выполняются в фоне.

## Как остановить?

Чтобы остановить плагин, нажмите кнопку "Остановить сервер" в веб-интерфейсе или просто закройте окно терминала, где запущен плагин.

## Для кого это?

Этот плагин будет полезен:

- **Разработчикам плагинов** — чтобы понять, какие возможности доступны
- **Пользователям системы** — чтобы увидеть, что могут делать плагины
- **Изучающим систему** — чтобы на практике познакомиться с функциональностью

## Безопасность

Все действия плагина выполняются в изолированной среде. Плагин имеет доступ только к разрешенным ресурсам:

- Только к указанным директориям файловой системы
- Только к разрешенным сетевым адресам
- Только к разрешенным системным командам
- Только к разрешенным переменным окружения

Это гарантирует безопасность выполнения плагина на вашей системе.
